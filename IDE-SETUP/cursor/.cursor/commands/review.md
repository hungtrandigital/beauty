# review

Use **Review Mode** for quality assurance and validation.

## Steps
1) Read `INDEX.md` for structure
2) Follow `0-agents/mode/review.md`
3) For code: follow `0-agents/agents/code-reviewer.md`
4) Check tests, security, performance, a11y, standards
5) Document findings and verdict (APPROVED/REJECTED/MINOR_CHANGES)

## Output locations
- Review comments (PR/comments) or `3-technical/3.2-implementation/reviews/`
- `8-governance/quarterly-retrospective/` (if retrospective)
- `8-governance/risk-register.md` (if risks)
- `8-governance/changelog.md` (if needed)

## Handoff
Use the orchestration format in `0-agents/mode/review.md` (`Current mode: review`, verdict, summary, blockers).
