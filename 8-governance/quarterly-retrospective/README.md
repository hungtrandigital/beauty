# Quarterly Retrospectives

## Purpose

This directory contains quarterly retrospectives documenting lessons learned, improvements, and team reflections. Retrospectives help identify what went well, what didn't, and how to improve.

## Retrospective Structure

Each retrospective follows this structure:
- **Period:** Quarter and year
- **Participants:** Team members involved
- **What Went Well:** Successes and achievements
- **What Didn't Go Well:** Challenges and issues
- **Action Items:** Improvements to implement
- **Metrics:** Key performance indicators

## Retrospective Template

Use the following template for new retrospectives:

```markdown

## Period
**Quarter:** Q[X] [Year]  
**Date Range:** [Start Date] - [End Date]  
**Participants:** *List of participants*

## What Went Well

- *Achievement 1*
- *Achievement 2*
- *Achievement 3*

### Success Metrics
- *Metric 1: Value*
- *Metric 2: Value*
- *Metric 3: Value*

### Highlights
- *Highlight 1*
- *Highlight 2*

## What Didn't Go Well

### Challenges

- *Challenge 2 - Impact*
- *Challenge 3 - Impact*

### Issues
- *Issue 1*
- *Issue 2*

### Missed Goals

- *Goal 2 - Why it was missed*

- *Lesson 1*
- *Lesson 2*

### Process Lessons
- *Lesson 1*
- *Lesson 2*

### Team Lessons
- *Lesson 1*
- *Lesson 2*

## Action Items

- [ ] *Action 3 - Owner - Due Date*

### Long-term Improvements
```
